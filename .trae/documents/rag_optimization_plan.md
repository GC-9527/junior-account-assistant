# 初会RAG知识库完善计划

## 一、现状分析

### 当前问题
1. **检索精度不足**：部分问题检索相似度偏低（40-60%）
2. **知识覆盖率不够**：部分常见问题无法回答（如个人所得税、应收账款与应付账款区别）
3. **仅使用向量检索**：缺乏关键词检索能力，对于明确概念的检索效果不佳
4. **元数据不完善**：文件分类和标签体系不够精细
5. **分块规则待优化**：切分策略需要调整以提高检索精度

### 测试结果总结
| 问题 | 结果 | 原因分析 |
|-----|------|---------|
| 固定资产折旧方法 | ✅ 成功 | 多意图拆分有效 |
| 什么是会计分录 | ✅ 成功 | 知识库有相关内容 |
| 应收账款和应付账款区别 | ❌ 失败 | 知识库内容不足 |
| 交易性金融资产做账 | ✅ 成功 | 分录大全匹配 |
| 个人所得税税率 | ❌ 失败 | 经济法内容不足 |

## 二、优化方案

### 1. 混合检索实现（优先级：高）

**目标**：结合BM25关键词检索和向量检索，设置动态权重

**实现步骤**：
- 安装 jieba 和 rank_bm25 库
- 创建 BM25 检索器模块
- 实现混合检索策略
- 设置权重调整机制（根据检索结果动态调整）

### 2. 元数据完善（优先级：高）

**目标**：优化文件分类和标签体系

**当前元数据结构**：
```json
{
  "book_type": "初级会计实务 / 经济法基础",
  "chapter": "第一章 资产",
  "knowledge_type": "概念定义 / 会计分录 / 税法法条",
  "exam_level": "必考 / 高频 / 了解",
  "source": "2025官方教材"
}
```

**优化方向**：
- 细化 chapter 字段（精确到小节）
- 增加 sub_topic 字段（子知识点）
- 完善 exam_level 标签（根据真题频率标注）
- 添加 difficulty 字段（难度等级）
- 优化 source 字段（统一命名规范）

### 3. 分块规则优化（优先级：高）

**目标**：调整切分策略提高检索精度

**当前规则**：
- 单块字符长度：500-800字
- 重叠上下文：80-120字

**优化方向**：
- 根据内容类型动态调整块大小：
  - 概念定义：300-500字
  - 会计分录：200-300字（保持分录完整性）
  - 真题习题：完整题目作为一块
  - 表格内容：整块保留
- 智能分隔符识别
- 知识点边界检测
